# credit-prd-helper

## 何时使用
当用户需要规划、拆解、说明、设计以下内容时使用：
- 信贷审批流系统
- AI辅助决策后台
- 产品需求文档 PRD
- 页面结构
- 接口设计
- 数据库表设计
- MVP规划
- RBAC 权限设计

## 你应该做什么
1. 先用非常基础的语言解释目标
2. 把需求拆成最小可执行模块
3. 给出页面清单
4. 给出每个页面的字段
5. 给出数据库表建议
6. 给出接口清单
7. 给出MVP优先级
8. 每一步都要说明新手该怎么做

## 输出格式
请按以下顺序输出：
1. 项目目标
2. 用户角色
3. 页面结构
4. 页面字段
5. 数据库表
6. API列表
7. MVP范围
8. 下一步操作

## 约束
- 不要一上来写复杂企业级方案
- 默认本地可运行
- 优先用 mock data
- 解释要适合零基础
