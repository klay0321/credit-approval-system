# 信贷风控审批流与 AI 辅助决策系统

一个适合本地运行和作品集展示的前端演示项目，围绕“信贷审批流 + 风控判断 + AI 辅助决策”场景，搭建了一个简化版的 B 端后台系统。

---

## 1. 项目背景

在真实的金融业务里，一笔贷款申请通常不会直接通过，而是要经历：

1. 客户提交申请
2. 风控系统评估风险
3. AI 给出辅助建议
4. 审批人员进行人工判断
5. 系统记录审批过程和结果

这个项目就是基于上面的业务流程，做出的一个本地演示版后台系统。

它不是一个真实银行系统，也不连接真实征信、真实审批引擎，而是先用 `mock data` 把页面流程、角色权限和审批交互跑通，方便后续继续扩展。

---

## 2. 项目目标

这个项目的目标不是一开始就做成复杂企业系统，而是先完成一个“最小可执行版本”，让项目具备以下能力：

- 能本地启动和运行
- 能展示审批后台页面
- 能通过 mock 登录模拟不同角色
- 能按角色展示不同任务范围
- 能查看审批任务详情
- 能模拟审批动作和审批结果展示
- 能展示审批历史记录
- 能展示工作台首页

对于 0 基础学习者来说，这个项目更重要的意义是：

- 理解一个 B 端后台项目是怎么拆页面的
- 理解“审批流”和“角色权限”这种常见业务逻辑
- 理解 mock data 驱动开发的基本思路

## 7. Mock 登录说明

当前项目没有接入真实账号密码系统，而是采用最简单的 mock 登录方式。

实现逻辑：

- 登录页中选择一个角色
- 系统把当前角色和 mock 用户名保存到浏览器 `localStorage`
- 后续页面从 `localStorage` 读取角色信息
- 如果没有登录信息，则自动跳转回 `/login`

这样做的好处是：

- 开发速度快
- 适合前期页面联调
- 适合 0 基础学习者先理解权限逻辑

## 10. 当前限制

当前版本仍然是演示版，所以有一些明确限制：

- 没有接真实数据库
- 没有接 Prisma / SQLite
- 没有接真实用户系统
- 没有接真实接口服务
- 没有接真实 AI 模型
- 审批动作目前只保存在前端本地状态中
- 页面中仍有少量中文编码问题需要继续清理

也就是说：

当前项目更像一个“前端原型 + 业务流程演示系统”，而不是正式生产系统。
